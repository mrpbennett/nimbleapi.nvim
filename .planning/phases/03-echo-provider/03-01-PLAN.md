**queries/go/echo-routes.scm** — Two patterns in one file:

Pattern 1 — Method shortcuts (ECHO-02, ECHO-04): `e.GET("/path", handler)`, `e.POST(...)`, `e.Any(...)`, etc.
```scheme
; Echo method shortcuts: e.GET("/path", handler), e.POST(...), e.Any(...), etc.
; Lua side filters @http_method against ECHO_METHODS table (GET/POST/PUT/DELETE/PATCH/HEAD/OPTIONS/CONNECT/TRACE/Any)
(call_expression
  function: (selector_expression
    operand: (identifier) @router_obj
    field: (field_identifier) @http_method)
  arguments: (argument_list
    (interpreted_string_literal) @route_path
    .
    (_) @func_name)) @route_def
```
No `#eq?` predicate on method — let all match and filter in Lua via ECHO_METHODS table. This is the same robust approach used for Gin.

Pattern 2 — Add (ECHO-03): `e.Add("GET", "/path", handler)`
```scheme
; Echo Add: e.Add("METHOD", "/path", handler)
; @_add_method captures the first string arg (the HTTP method)
; @http_method captures "Add" — Lua side checks this to switch to @_add_method
(call_expression
  function: (selector_expression
    operand: (identifier) @router_obj
    field: (field_identifier) @http_method
    (#eq? @http_method "Add"))
  arguments: (argument_list
    (interpreted_string_literal) @_add_method
    (interpreted_string_literal) @route_path
    .
    (_) @func_name)) @route_def
```

**queries/go/echo-groups.scm** — One pattern (ECHO-05, ECHO-06):

Copy gin-groups.scm exactly — the `e.Group("/prefix")` call has the identical Go AST shape as Gin's `router.Group("/prefix")`. Only the variable names in real code differ; the query structure is the same.

```scheme
; Echo Group variable: g := e.Group("/api") or g := e.Group("/api", middleware)
; @group_var   — the new group variable name (e.g., "api", "v1")
; @router_obj  — the parent Echo/Group variable (e.g., "e", "api")
; @route_path  — the prefix path string (e.g., "/api")
(short_var_declaration
  left: (expression_list
    (identifier) @group_var)
  right: (expression_list
    (call_expression
      function: (selector_expression
        operand: (identifier) @router_obj
        field: (field_identifier) @_group_method
        (#eq? @_group_method "Group"))
      arguments: (argument_list
        (interpreted_string_literal) @route_path)))) @route_def
```

IMPORTANT notes for the executor:
- Use `(field_identifier)` (NOT `(identifier)`) for method names in selector_expression
- Use `(_) @func_name` to capture any handler expression (identifiers, selector_expressions like `h.Handler`, func literals)
- The `.` (anchor) before `(_) @func_name` in Pattern 1 ensures the handler is the second argument; in Pattern 2 it ensures handler is the third
- Do NOT anchor the group arguments after `@route_path` — middleware args may follow: `e.Group("/prefix", authMiddleware)`
- DO use `#eq?` for "Add" and "Group" since those are exact match filters
- Echo method names are ALL CAPS: GET, POST, PUT, DELETE, PATCH, HEAD, OPTIONS, CONNECT, TRACE (plus `Any`)

**3. Add resolve_prefix helper** (copy from gin.lua — identical logic):
```lua
local function resolve_prefix(var_name, groups, visited)
  if not groups[var_name] then return "" end
  if visited[var_name] then return "" end  -- cycle guard
  visited[var_name] = true
  local entry = groups[var_name]
  local parent_prefix = resolve_prefix(entry.parent, groups, visited)
  return parent_prefix .. entry.prefix
end
```

**4. Implement extract_routes(filepath)** — Two-pass algorithm:

Pass 1: Run echo-groups.scm query to collect group variable table:
- Parse file: `local root_node, source = parser.parse_file(filepath, "go")`
- Load query: `local ok, groups_query = pcall(parser.get_query_public, "echo-groups", "go")`
- For each match, extract captures: `@group_var` -> var name, `@router_obj` -> parent var, `@route_path` -> prefix string
- Strip quotes from `@route_path` text
- Store: `groups[group_var_text] = { prefix = stripped_path, parent = router_obj_text }`

Pass 2: Run echo-routes.scm query to extract routes:
- Load query: `local ok2, routes_query = pcall(parser.get_query_public, "echo-routes", "go")`
- For each match, extract captures
- Node extraction: `local node = type(nodes) == "table" and nodes[1] or nodes` (CRITICAL — Neovim 0.10+ compat)
- For `@http_method`: if text is `"Add"`, read method from `@_add_method` capture (strip quotes, uppercase); otherwise look up in ECHO_METHODS
- For `@route_path`: strip quotes
- For `@func_name`: **CRITICAL** — check `node:type() == "func_literal"` first; if true, set `func_name_text = ""`; otherwise call `parser.get_text(node, source)` (prevents multi-line body crash from Gin bug #1)
- For `@router_obj`: get text
- For `@route_def`: `local row, _, _, _ = node:range(); route_def_line = row + 1`
- After processing: `prefix = resolve_prefix(router_obj_text, groups, {})`
- `full_path = normalize_path(prefix .. route_path_text):gsub("//+", "/")`
- If `full_path == ""` then `full_path = "/"`
- Build record: `{ method = method, path = full_path, func = func_name_text, file = filepath, line = route_def_line or 0 }`
- Sort by line before returning

**5. Implement get_all_routes(root)**:
```lua
function M.get_all_routes(root)
  local go_files = utils.glob_files(root, "**/*.go", {
    "vendor", "testdata", "node_modules", ".git",
  })
  local all_routes = {}
  for _, f in ipairs(go_files) do
    if utils.file_contains(f, "echo.")
      or utils.file_contains(f, ".GET(")
      or utils.file_contains(f, ".POST(")
      or utils.file_contains(f, ".Group(")
      or utils.file_contains(f, ".Add(")
    then
      local routes = M.extract_routes(f)
      for _, route in ipairs(routes) do
        table.insert(all_routes, route)
      end
    end
  end
  return all_routes
end
```

**6. Implement get_route_tree(root)** (thin wrapper):

```lua
function M.get_route_tree(root)
  local routes = M.get_all_routes(root)
  if not routes or #routes == 0 then return nil end
  return { file = "", var_name = "EchoApp", routes = routes, routers = {} }
end
```
