**1. Create queries/go/echo-testclient.scm** (ECHO-07):

Echo test files call `httptest.NewRequest(method, "/path", body)` where:
- Package is `httptest` (from `net/http/httptest`) — NOT `http`
- Function is `NewRequest`
- First argument is the HTTP method — can be a string literal `"GET"` OR a constant `http.MethodGet` (selector_expression)
- Second argument is the path string literal

To handle both method argument forms, capture the first argument as `(_)` (any node):

```scheme
; httptest.NewRequest(method, "/path", ...) calls in Echo test files
; method can be a string literal ("GET") or http.MethodXxx constant (selector_expression)
(call_expression
  function: (selector_expression
    operand: (identifier) @_pkg
    field: (field_identifier) @_func
    (#eq? @_pkg "httptest")
    (#eq? @_func "NewRequest"))
  arguments: (argument_list
    (_) @http_method_expr
    (interpreted_string_literal) @test_path
    .)) @test_call
```

Capture names:
- `@_pkg` and `@_func` — filtered by `#eq?` predicates (underscore prefix = internal, not consumed by Lua)
- `@http_method_expr` — the method argument node (can be string literal or selector_expression)
- `@test_path` — the path string (second argument)
- `@test_call` — full call expression node (for line number)

Note: Using `(_) @http_method_expr` instead of `(interpreted_string_literal) @http_method` to capture BOTH `"GET"` and `http.MethodGet` forms. The Lua code must handle both node types.

**2. Implement extract_test_calls_buf(bufnr) in echo.lua:**

Replace the existing stub. Key differences from gin.lua version:
- Query name is `"echo-testclient"` (not `"gin-testclient"`)
- Method capture name is `"http_method_expr"` (not `"http_method"`)
- Method text needs type-aware processing: string literal vs selector_expression (http.MethodGet form)

```lua
function M.extract_test_calls_buf(bufnr)
  local root_node, buf = parser.parse_buffer(bufnr, "go")
  if not root_node then
    return {}
  end

  local filepath = vim.api.nvim_buf_get_name(bufnr)

  local ok, query = pcall(parser.get_query_public, "echo-testclient", "go")
  if not ok or not query then
    return {}
  end

  local calls = {}
  for _, match, _ in query:iter_matches(root_node, buf, 0, -1) do
    local call = { file = filepath }
    for id, nodes in pairs(match) do
      local name = query.captures[id]
      local node = type(nodes) == "table" and nodes[1] or nodes
      local text = parser.get_text(node, buf)

      if name == "http_method_expr" then
        -- Handle both: "GET" (string literal) and http.MethodGet (selector_expression)
        local node_type = node:type()
        if node_type == "interpreted_string_literal" then
          call.method = strip_quotes(text)
        elseif node_type == "selector_expression" then
          -- http.MethodGet -> extract "MethodGet" -> strip "Method" prefix -> uppercase "GET"
          local method_suffix = text:match("^%w+%.Method(.+)$")
          if method_suffix then
            call.method = method_suffix:upper()
          else
            -- Fallback: use the full text uppercased (handles edge cases)
            call.method = text:upper()
          end
        end
      elseif name == "test_path" then
        call.path = strip_quotes(text)
      elseif name == "test_call" then
        call.line = node:range() + 1
      end
    end

    if call.method and call.path then
      table.insert(calls, call)
    end
  end

  return calls
end
```

Key details:
- Uses `parser.parse_buffer(bufnr, "go")` (NOT parse_file — live buffer)
- Second return from parse_buffer is the bufnr (used as source for get_text)
- `http_method_expr` capture handles string literals and http.MethodXxx constants
- `method_suffix:upper()` ensures consistent uppercase regardless of input case
- Line is 1-indexed (node:range() returns 0-indexed row + 1)
- Per D-04: codelens shows first match only (handled in codelens.lua, not here)
